Chapter 01

引言 — 为什么需要这份文档

CycleMind 利用多 AI Agent 协作技术，将自然语言需求自动转化为架构图、ER 图、API 规范和开发计划，大幅缩短从需求到设计的周期。

📚

项目背景

传统 SDLC 中需求分析与架构设计耗时较长，高度依赖资深工程师经验。CycleMind 将这一过程从数天压缩至分钟级别。

🎯

文档目的

为开发团队、测试团队、项目管理者和最终用户提供完整、准确的需求描述，作为设计、编码和验收的基准。

🔗

系统关系

独立 Web 应用，通过 DeepSeek API 获取 AI 能力，Neon PostgreSQL 存储数据，部署于 Vercel 平台。

Chapter 03

用户需求与业务场景

三种角色、五条业务流程、十一项用户需求

核心角色

💻 开发者

输入自然语言需求，使用 AI 生成全套设计文档，编辑和导出最终成果。

核心角色

🛠 架构师

审阅 AI 生成的设计方案，调整架构图和 API 规范，管理文档版本和技术偏好。

核心角色

📊 产品经理

提出业务需求，使用模板快速启动项目，在仪表盘总览项目状态。

核心业务流程

1. 用户入驻

邮箱密码注册 / GitHub OAuth 一键登录 / 个人资料管理

2. 项目管理

创建项目、编辑信息、归档删除、模板快速创建

3. AI 辅助设计

对话式需求采集 → 智能编排 → 多 Agent 并行生成 → 文档交付

4. 文档迭代

查看文档 → 手动编辑 → 追加需求 → 增量生成新版本

Chapter 04

功能需求 — 十大功能模块

涵盖认证、项目管理、AI 对话、Agent 编排、文档生成等完整链路

F-01

用户注册

邮箱 + 密码（8-100 位，含大小写和数字）或 GitHub OAuth。bcrypt 加密存储，自动登录。

F-02

用户登录

Credentials 或 GitHub OAuth 双通道。JWT 无状态令牌，中间件自动保护路由。

F-03

项目管理 CRUD

创建 / 查看 / 编辑 / 删除项目。支持模板快速创建，级联删除关联数据。

F-04

对话式需求采集

对话 Agent（temp=0.7）多轮引导用户细化需求。SSE 流式推送，检测 [READY] 自动触发编排。

F-05

Agent 编排

编排器（temp=0.1）分析需求，返回 Agent 组合。用户可勾选/取消，异常时降级到默认方案。

F-06

多 Agent 并行生成

5 个专业 Agent 通过 Promise.all 并行执行。SSE 实时推送进度，append-only 版本控制。

F-07

文档预览与编辑

5 个标签页切换。Mermaid 图表支持缩放平移，编辑模式保存自动创建新版本。

F-08 / 09 / 10

模板 · 用户画像

5 个内置项目模板。user.md 个人上下文（AI 自动引用）。
